# ADR-023: Порог drawdown kill-switch — 5% основной трек, до 15% для альт-стратегий

**Статус:** ACCEPTED
**Дата:** 2026-06-22
**Владелец:** Yurii (Owner)

---

## Контекст

Аудит кода (`CODE_AUDIT_BACKLOG.md`, AUD-01) выявил рассогласование порога
аварийной остановки (kill-switch) по просадке портфеля:

- `RiskConfig.max_drawdown_stop = 0.05` (`spa_core/risk/policy.py`), CLAUDE.md и
  `GoLiveChecker` (`golive_checker.py`, блок go-live при `drawdown >= 5%`)
  определяли стоп как **5%**.
- Фактически активный в дневном цикле автономный kill-switch
  (`spa_core/governance/kill_switch.py`, `run_kill_switch_check`, вызывается из
  `cycle_runner.py`) срабатывал только при **15%** просадки, причём со
  строгим сравнением `>` (ровно 15.00% не триггерило).
- `RiskPolicy.check_portfolio_health` (5%) в основном цикле не вызывался, а
  проверка новой позиции строила свежий пустой `PortfolioState` → её drawdown
  всегда равнялся 0.

**Итог:** документированный 5%-стоп на живом треке фактически не действовал;
единственным активным был 15%-порог. Портфель мог просесть на 5–15% —
заявленную «зону закрытия всего» — без срабатывания перехода в кэш.

## Решение

**Основной трек: kill-switch по просадке срабатывает при `drawdown >= 5%`.**
Сравнение изменено со строгого `>` на `>=` (ровно на пороге — срабатывает).
Источник значения по умолчанию приведён в соответствие с
`RiskConfig.max_drawdown_stop` (5%).

**Альтернативные более рисковые стратегии** (тестовый режим) могут поднимать
порог **до 15%** (`DRAWDOWN_ALT_MAX_PCT`). Это осознанное расширение для
экспериментальных/спекулятивных стратегий, которым свойственна большая
волатильность, при условии что они изолированы от основного капитального трека.

### Механика

`KillSwitchChecker.__init__(..., drawdown_threshold_pct: float | None = None)`:

1. Приоритет источников порога: явный аргумент → `data/risk_policy.json`
   (ключ `DRAWDOWN_KILL_THRESHOLD_PCT`) → compile-time дефолт `5.0`.
2. Значение клампится в диапазон `[0, DRAWDOWN_ALT_MAX_PCT]` = `[0, 15]`.
   **Никакой агент/стратегия не может ослабить порог выше 15%** — это жёсткий
   потолок (в духе FORBIDDEN: `approved=False` от kill-switch неоспорим).
3. `run_kill_switch_check(..., drawdown_threshold_pct=None)` пробрасывает порог;
   основной цикл (`cycle_runner`) вызывает без аргумента → **5%**.

### Константы (`spa_core/governance/kill_switch.py`)

| Константа | Значение | Назначение |
|---|---|---|
| `DRAWDOWN_THRESHOLD_PCT` | `5.0` | основной трек (= `RiskConfig.max_drawdown_stop`) |
| `DRAWDOWN_ALT_MAX_PCT` | `15.0` | верхняя граница для альт-стратегий (clamp) |

## Последствия

- **+** Документированный 5%-стоп теперь реально действует на основном треке.
- **+** Единая, явная семантика порога; альт-стратегии параметризуемы, но не
  могут выйти за 15%.
- **−** На основном треке kill-switch будет срабатывать чаще (раньше — только
  при 15%). Это намеренно: приоритет — защита капитала.
- Затронуты: `kill_switch.py` (константы, конструктор, `check_drawdown_trigger`,
  `run_kill_switch_check`), тесты `test_kill_switch.py` / `test_kill_switch_sharpe.py`.
- `RiskPolicy.version` **не меняется** (остаётся `v1.0`) — правка касается
  governance-слоя kill-switch, а не риск-конфига; смысловой порог
  `max_drawdown_stop=0.05` уже был в `RiskConfig` и теперь лишь корректно
  применяется.
