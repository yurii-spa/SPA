# ADR-TEST · Smoke-test контура владельца (ENV_SETUP_BRIEF_v3, Этап 8)

- **Статус:** Accepted
- **Дата:** 2026-07-15
- **Автор/утвердил:** владелец (перевёл TEST-карточку в Owner Done) + оркестратор (инжест)

## Контекст

ENV_SETUP_BRIEF_v3 построил files-first контур владельца (Owner Decisions + Inbox трекеры,
протокол оркестратора, наблюдение, Obsidian-база знаний). Перед отключением старого механизма
`OWNER_DECISIONS_NEEDED.md` бриф (Этап 8) требует сквозного smoke-теста реального пути:
уведомление → действие владельца → инжест оркестратором.

## Решение

Прогнан полный цикл, все звенья подтверждены вживую:
1. Оркестратор создал TEST `needs-owner` карточку + Telegram-уведомление (§3.3) — **доставлено**.
2. Владелец перевёл карточку в `owner-done` в Nimbalyst — **Nimbalyst записал статус обратно в файл**
   (files-first writeback подтверждён).
3. Владелец отправил боту **голосовое** — офлайн whisper-расшифровка → Inbox-карточка `source: voice`
   с корректным текстом (+ `/task`-путь тоже проверен ранее).
4. Оркестратор: owner-done → `ingested` (+ этот ADR-TEST) + обновление STATE/journal; inbox-задания
   классифицированы и декомпозированы в задачи с acceptance-критериями (§6.3/6.4).

Контур принят как операционный. Старый `OWNER_DECISIONS_NEEDED.md` остаётся указателем (retired).

## Последствия

- ✅ Полный путь владельца работает: карточки в git ⇄ Nimbalyst kanban ⇄ Telegram (текст+голос) ⇄ оркестратор.
- ⚠️ Известное ограничение Nimbalyst: новые карточки-файлы видны только после переоткрытия workspace
  (индексирует при open, не следит в реальном времени). Живой сигнал — Telegram; Nimbalyst — доска, освежается при открытии.
- Источник правды — файлы в git; инструмент может умереть, очередь останется.
