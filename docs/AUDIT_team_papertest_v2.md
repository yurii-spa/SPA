## T7: Team Tab

### Текущее содержимое

**Расположение в файле:** строки 1368–1720 (**352 строки**)  
**Кнопка навигации:** `👾 Team` (7-я позиция)

Таб состоит из шести блоков:

**1. Заголовок — "Операционный центр агентов"** (Agent Operations Center v1.6)  
Хардкодированный баннер с живой точкой "PAPER TRADING АКТИВЕН".

**2. Daily Commentary** (читает `data/agent_summaries.json`)  
Две карточки — Trader Agent Commentary и Risk Agent Commentary.  
Файл существует, последнее обновление: `2026-06-10T08:56:41`. Поле `used_llm: false` —
комментарий сгенерирован детерминированно, не LLM-агентом. Содержимое минимальное и stale.

**3. Четыре агент-карточки** (Data Agent, Strategy Agent, Risk Monitor, CEO Agent)  
Каждая карточка показывает: имя, роль, статус, задачу, расписание, последнее действие, кнопку "Подробнее".

Источники данных:
- Статусы (`АКТИВЕН`) — **жёстко захардкожены** в HTML; JS-код всегда выставляет `active` (строка 6901:
  `statusEl.className = 'ops-status active'; // always active in paper trading`).
- Описания задач (ops-task) — хардкодированные placeholder-тексты.
- Время последнего действия (`ops-lasttime`) — подтягивается из `data/agent_runtime_log.json`
  (файл существует, обновляется автоматически).
- Модали "Подробнее" — читают `window._spa_status` (paper_trading_status.json).

**4. Архитектурная диаграмма** — SVG с анимированными потоками между нодами.  
Показывает: CEO Agent (Claude Sonnet 4.6), Data Agent (Gemini Flash-Lite), Strategy Agent (Gemini 2.5 Flash),
Risk Monitor (Claude Haiku 4.5), Execution, Blockchain. **Визуально красиво, но фактически устарела** —
отображает агентную архитектуру с LangGraph/Gemini, которая была до teardown (2026-06-10).
Реальный runtime — `cycle_runner` на Python stdlib, без LangGraph и Gemini.

**5. Технический стек и Роадмап** — хардкодированные таблицы.  
Стек содержит несоответствия реальному состоянию:
- "LangGraph (5 нод)" — не используется в актуальном runtime.
- "SQLite + WAL" — не используется; данные в JSON-файлах.
- "CI/CD: GitHub Actions (4h)" — неверно; реальный runtime: launchd `com.spa.daily_cycle` (08:00).
- "AG-UI / CopilotKit — планируется v0.5" — отсутствует в MASTER_PLAN.

Роадмап: v2.0 Go-Live указан как `~2026-07-15`, но ADR-002 перенёс на `~2026-08-01`.

**6. SSE Stream лента ("📡 STREAM АГЕНТОВ — РЕАЛЬНОЕ ВРЕМЯ")**  
Показывает пустой placeholder: «Нет событий · запустите `uvicorn spa_core.api.server:app --port 8765`».  
Требует FastAPI-сервер на localhost:8765, который в штатном launchd-режиме **не запущен**.
SSE подключение (`_sseConn`) не устанавливается → лента всегда пустая.

### Оценка ценности

| Критерий | Оценка |
|---|---|
| Реальные данные прямо сейчас | Частично: `ops-lasttime` из agent_runtime_log.json, но статусы hardcoded |
| Нужен solo-оператору | Низкая: solo-оператор и так знает статус системы по Go-Live / System табам |
| Дублирует что-то другое | Частично: информация о позициях/риске есть на Paper Test; архитектура — нигде больше |
| Будет нужен в будущем | Да: для питча инвесторам и партнёрам диаграмма + стек ценны |
| Критические ошибки в данных | Да: архитектура, стек, CI/CD строка, go-live дата — всё устарело |

### Вердикт: **Реструктурировать**

Таб имеет сильный визуальный каркас (диаграмма, карточки), который ценен для внешней аудитории.
Но текущее содержимое создаёт **ложное представление об архитектуре** и будет вводить в заблуждение инвесторов.

### Что изменить

1. **Архитектурная диаграмма** — заменить/обновить: убрать LangGraph, Gemini-агентов;
   показать реальную цепочку `launchd → cycle_runner → adapters → StrategyAllocator → RiskPolicy → trades.json`.
   
2. **Технический стек** — исправить:
   - Оркестрация: `launchd (daily 08:00)` вместо `LangGraph (5 нод)`
   - База данных: `JSON файлы (stdlib)` вместо `SQLite + WAL`
   - CI/CD: `launchd com.spa.daily_cycle` вместо `GitHub Actions (4h)`
   - Убрать строку "AG-UI / CopilotKit"

3. **Go-Live в роадмапе** — исправить `~2026-07-15` → `~2026-08-01` (ADR-002).

4. **Статусы агент-карточек** — заменить hardcoded "АКТИВЕН" на реальный статус из
   `agent_runtime_log.json` (есть ли запись за последние 24h).

5. **SSE Stream лента** — скрыть через `display:none` пока FastAPI не является штатным runtime;
   заменить на `agent_runtime_log.json` (файл уже существует и обновляется).

6. **agent_summaries.json** — обеспечить обновление в ходе ежедневного цикла
   (сейчас дата: 2026-06-10, stale на 2 дня).

## T8: Paper Test Tab (id: tab-dashboard)

### Текущее содержимое

**Расположение в файле:** строки 1184–1350 (**166 строк**)  
**Кнопка навигации:** `📑 Paper Test` (2-я позиция, с подсказкой "Paper Trading")

Таб — главный финансовый дашборд. Содержит 10 блоков:

1. **Заголовок** с health badge и кнопками "Обновить / PDF".
2. **Paper Status Strip** — показывает режим из `paper_trading_status.json` (is_demo: true/false → DEMO / LIVE DATA).
3. **4 метрики**: total capital, deployed, cash buffer, unrealized PnL — из `current_positions.json`.
4. **Open Positions** — таблица позиций (Protocol, Amount, APY, PnL) — из `current_positions.json`.
5. **Risk секция**: health, VaR 95%/7d, drawdown, alerts, trades — из `paper_trading_status.json`.
6. **8-week clock** — прогресс-бар трека.
7. **APY Gap Tracker** — визуальный трекер текущего APY vs цели 7.30%; с рычагами Pendle PT и Sky/sUSDS.
8. **Portfolio Chart** — equity curve из `data/equity_curve_daily.json` (реальный трек с 2026-06-10).
9. **APY Chart** — live APY всех протоколов по цветовой кодировке (позиция / T1 без позиции / T2 без позиции).
10. **Алерты, История сделок, Strategy Race** — из `alerts.json`, `trades.json`.

**Источники данных (все реальные):**
- `data/current_positions.json` ✓
- `data/paper_trading_status.json` ✓
- `data/equity_curve_daily.json` ✓
- `data/trades.json` ✓
- DeFiLlama feed (через JSON-кэш) ✓

Таб загружается лениво при первом переходе (`loadDashboard()`), а также принудительно при изменении
paper_trading_status (`loadDashboard()` на строке 6377).

### Вердикт: **Реструктурировать (переименовать)**

Таб работает правильно и содержит наиболее ценные данные в дашборде. Единственная проблема — неудачное название.

### Что изменить

1. **Переименовать кнопку**: `📑 Paper Test` → `💰 Portfolio` (или `📊 Trading`).  
   Текущее название "Paper Test" звучит как временный артефакт разработки, а не основной финансовый монитор.  
   Внутренний id `tab-dashboard` и JS можно оставить без изменений — это не видно пользователю.

2. **Footer строка** (строка 1348): `SPA Paper Trading v0.4 · обновляется каждые 4ч` —  
   исправить на `обновляется ежедневно в 08:00` (реальное расписание launchd).

3. **Опционально**: добавить "DEMO" / "LIVE" бейдж в название кнопки навигации
   на основе `paper_trading_status.json.is_demo`, чтобы режим был виден без открытия таба.
