# SPA — Risk-tier пакеты (для инвестора)

> Три уровня доходности под разный аппетит к риску. Каждый пакет — это стратегии, прошедшие
> **Tier-1 валидацию** (как в институциональных квант-фондах), а не сырой рейтинг бэктеста.
> Сейчас paper trading. Цифры — результат на реальных исторических данных DeFiLlama, не обещание.
> Live: **earn-defi.com/packages** · API: **api.earn-defi.com/api/tier1/packages**

## Три пакета

| Пакет | Target net APY | Лимит просадки | Для кого |
|---|---|---|---|
| 🟢 **Conservative** | 2–6% | ≤ 3% | Сохранение капитала, blue-chip lending (T1) |
| 🟡 **Balanced** | 6–12% | ≤ 10% | Баланс доходность/риск, T1+T2 |
| 🟣 **Aggressive** | 12%+ | ≤ 25% | Максимум доходности, T2/T3, диверсификация |

## Что значит «Tier-1 validated»

Стратегия попадает в пакет, только если проходит ВСЕ проверки:

1. **Реальные данные** — бэктест на исторических APY DeFiLlama (не mock, не шум), с проверкой
   целостности (нет lookahead/будущих дат, нет аномалий).
2. **Net-of-cost APY** — доходность ПОСЛЕ газа, slippage, bridge — то, что реально остаётся.
3. **Out-of-sample hold** — доходность держится на held-out периоде (последние 30%), а не только
   на исторически высокодоходном прошлом.
4. **Capacity** — пул достаточно глубок (позиция ≤ 2% TVL) — стратегия масштабируется на AUM.
5. **Tail-risk** — учтён риск потери принципала (депег/эксплойт) по tier протокола → risk-adjusted APY.
6. **Стресс** — worst-case под сценариями (rate-collapse, депег, эксплойт).
7. **Диверсификация** — в пакете стратегии с низкой взаимной корреляцией (не одна ставка).

## Метрики на странице пакета

- **net APY** — доходность после издержек.
- **Risk-adjusted APY** — net APY минус ожидаемая потеря принципала (tail-risk).
- **Worst case (стресс)** — годовой результат в худшем из сценариев.
- **Capacity / scaling** — макс. размер фонда; бейджи 1M/10M/100M = выдерживает ли этот AUM.
- **Worst drawdown** — макс. историческая просадка.

## Честность

Если в пакете «нет валидированных стратегий» — значит ни одна не прошла планку Tier-1 на текущих
данных. Мы НЕ показываем непроверенные стратегии как готовый продукт. Tail/principal risk управляется
отдельно детерминированным RiskPolicy (LLM запрещён в риск-контуре).
