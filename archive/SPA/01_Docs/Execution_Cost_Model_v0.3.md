## 1. Назначение

Execution Cost Model определяет, **когда действие экономически оправдано, а когда запрещено**. Документ — обязательный шлюз перед любым on-chain действием.

---

## 2. Базовые принципы

- стоимость действия считается **до** его выполнения;
- учитываются **все** издержки, не только газ;
- отсутствие действия — валидный и часто предпочтительный результат;
- консерватизм > активность;
- модель не может быть обойдена ручным решением без изменения документа.

---

## 3. Категории издержек

### 3.1. Gas Cost
- вход, выход, approve, claim;
- rolling 7-day median + buffer +20%;
- в USD-эквиваленте на момент решения.

### 3.2. Protocol Fees
- deposit/withdrawal fees;
- performance fees (например, Yearn 15%);
- скрытые комиссии.

### 3.3. Slippage
- по глубине пула и размеру операции.

### 3.4. Opportunity Cost
- учитывается только как контекст;
- **никогда не разрешает действие**.

---

## 4. Execution Risk Buffer

20% от Total Execution Cost. Все цифры — в USD через USDT-эквивалент.

---

## 5. Total Execution Cost

```
Total Execution Cost (USD) =
    Gas Cost (USD)
  + Protocol Fees (USD)
  + Slippage (USD)
  + Execution Risk Buffer (= 20% от суммы выше)
```

---

## 6. Expected Benefit

- разница в net yield между текущим и целевым состоянием;
- на разумном временном горизонте;
- скорректирована на риск.

**Краткосрочный APY не является основанием для действия.**

---

## 7. Горизонт оценки

Для lending-стратегий: **7–14 дней**.

Если протокол часто меняет APR — использовать нижнюю границу.

---

## 8. Критерий разрешения

Действие разрешено только если:

```
Expected Benefit > Total Execution Cost × Safety Multiplier
```

### Safety Multiplier по типам:

| Тип операции | Safety Multiplier |
|--------------|-------------------|
| Плановый ребаланс между whitelisted-протоколами | **1.6** |
| Новый вход в whitelisted-протокол | **2.0** |
| Выход из whitelisted-протокола (плановый) | **1.4** |
| Экстренный выход из эксплуатируемого протокола | **0** (Risk Policy 7.1, не cost model) |
| Защитный выход (de-peg / liquidity drop) | **0.8** |

### 8.1. Cumulative cost limit
- сумма Execution Costs за rolling 7 дней ≤ **30%** от Realized Yield за тот же период по стратегии;
- превышение → block новых ребалансов до next reporting period.

### 8.2. Breakeven-формула (добавлено v0.4.5)

```
Минимальный ΔnetAPY (годовых) =
  (Total Execution Cost × Safety Multiplier) / (Capital × Holding Days / 365)
```

**Пример 1: $10K, 14 дней**
```
Total Cost = $25 × 2 × 1.2 (buffer) = $60
× Safety Multiplier 1.6 = $96
ΔnetAPY = $96 / ($10K × 14/365) = 25% годовых
```
**Вывод:** при $10K ребалансировки между Tier 1 — почти никогда. Стратегия: войти один раз, держать долго.

**Пример 2: $50K, 14 дней**
```
ΔnetAPY = $96 / ($50K × 14/365) = 5%
```
**Вывод:** оправданы при разнице ≥ 5% (например, между Tier 1 lending и Tier 2 Pendle PT).

**Пример 3: $250K, 14 дней**
```
ΔnetAPY = $96 / ($250K × 14/365) = 1%
```
**Вывод:** оправданы даже мелкие арбитражи. Auto-compound vault Yearn делает это автоматически — поэтому для $250K+ Yearn особенно эффективен.

### 8.3. Gas spike сценарий

При gas median × 5 от rolling 7d:

```
Cost per operation: $25 × 5 = $125
Total Execution Cost: $125 × 2 × 1.2 = $300
× Safety Multiplier 1.6 = $480
```

| Капитал | Holding 14 дней | Holding 30 дней |
|---------|----------------|------------------|
| $10K | 125% (всегда запрет) | 58% (всегда запрет) |
| $50K | 25% (запрет 99%) | 12% (редко) |
| $100K | 12.5% (редко) | 6% (возможно) |
| $250K | 5% (умеренно) | 2.4% (часто) |

**Действия при gas spike > 5×:** плановые операции откладываются; защитные выходы при SEV-1 исполняются без cost model.

**Действия при gas spike > 10×:** Panic gas regime — все операции, кроме экстренных выходов при SEV-1, запрещены.

## 9. Триггеры пересчёта

- изменение net yield;
- изменение газа;
- изменение ликвидности;
- изменение риск-статуса;
- изменение состояния стратегии.

Повторные пересчёты без изменения условий запрещены (защита от forcing решения).

### 9.1. Газ — единицы и расчёт
- единица: **USD-эквивалент**;
- источник ETH цены: oracle (Data & Signals 4.2);
- gas price: rolling 7-day median + buffer +20%;
- panic gas (> 10× median): см. Risk Policy 7.5.

---

## 10. Поведение в пограничных случаях

Если:
- данные неполные;
- расчёт неоднозначен;
- Expected Benefit / Total Cost между 1.0 и 1.2 от Safety Multiplier;

→ действие **не выполняется**.

«Близко к порогу» — само по себе сигнал нестабильности.
