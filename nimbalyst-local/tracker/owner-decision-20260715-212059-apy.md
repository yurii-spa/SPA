## Что случилось и почему это важно
Ты переслал алерт Site Custodian с двумя FAIL. Я разобрался:

**1) SNAPSHOT_BEHIND_API (apy=4.742 != API apy=6.654)** — это ЛОЖНОЕ срабатывание, НЕ ошибка на сайте.
Монитор сравнивает два РАЗНЫХ числа: на сайте показано стабильное «трековое» APY за весь период
(4.742%, считается от стартовой точки), а живой API отдаёт волатильное «сегодняшнее» APY (6.654%,
годовые из доходности одного дня — прыгает каждый день). Генератор снапшота НАМЕРЕННО берёт стабильное
число, а не дневное. Снапшот свежий и корректный (перегенерация даёт те же 4.742). Монитор зря их
сравнивает и будет ругаться почти каждый день.

**2) MISSING_ASOF** — на главной странице нет подписи «на дату …» рядом с числом. Это по делу: число
без даты выглядит непрозрачно. Безопасно добавить дату-подпись (это не меняет само число доходности).

Почему не чиню сам: правка касается публичного honesty/audit-механизма (инвариант #8) — по правилу #16
такие вещи не трогаю молча, а спрашиваю тебя.

## Что от тебя нужно
Реши по каждому (моя рекомендация первая):
- **FAIL #1:** (а, рекомендую) поправить монитор — убрать сравнение стабильного трекового APY с
  волатильным дневным (оставить проверку по дням трека + возрасту снапшота, они валидные); (б) оставить
  как есть и терпеть ежедневные ложные алерты; (в) доработать API, чтобы он отдавал такое же трековое
  APY для сравнения «как с как».
- **FAIL #2:** разрешить добавить подпись «на дату …» на главную (рекомендую — быстро и честнее).
