## Три причины остановки (без изменений, подтверждены данными)

**1. Diminishing returns — петля штампует почти-дубликаты.**
Цепочка v3.40 → v3.46 — это одна и та же фича в шести обёртках, по собственному признанию лога каждый раз «зеркалит предыдущий 1-в-1»:
`alert_apy_feed_stale` → `…_protocol_drop` → `…_tvl_drop` → `…_protocol_anomaly` → `…_schema_drift` → `…_protocol_staleness`.
Предложенный v3.47 («агрегированный health-бейдж» или «свести 6 алертов в один») — фактически уборка за самой петлёй. Размер `KANBAN.json` за сутки вырос ~113KB → ~176KB. Правило «Status pass запрещён» здесь генерирует работу ради работы.

**2. Нет безопасной HIGH-задачи для автопилота.**
Проверил KANBAN. Все HIGH-карточки — это либо **действия пользователя** (BL-004 GitHub Pages, BL-005/SPA-BL-008 Telegram, SPA-BL-007 RPC-ключи в Secrets, SPA-BL-009 Gnosis Safe + wallet), либо **live-capital фичи** (FEAT-001 «Real Capital Execution v2.0», FEAT-002 «Multi-Strategy Live Portfolio»). Ни то, ни другое нельзя автономно «брать в работу и пушить» без человека. Поэтому петля и выдумывает мониторинг-мелочь — реальный безопасный бэклог для неё закончился.

**3. Это money-moving код + утечка секрета.**
В системе есть `eth_signer.py`, `send_raw_transaction`-роутинг, live supply/withdraw в адаптерах. Авто-генерировать и **пушить непроверенный код в систему, подписывающую реальные транзакции, каждый час без человека** — неприемлемый риск. Плюс GitHub PAT (`ghp_…`) лежит открытым текстом в задании и прокидывается в URL `localhost:8765` — токен следует считать **скомпрометированным**.
