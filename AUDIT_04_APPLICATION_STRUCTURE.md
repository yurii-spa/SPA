## Frontend map — `landing/` (Astro)

- **30 top-level pages** (`landing/src/pages/*.astro`): `index`, `dashboard` (canonical app dashboard), `status`, `system`, `track-record`, `research`, `disclaimer`, `risk-disclosure`, `rates-desk`, `rwa-backstop`, `structural-desk`, `trust`, `methodology`, `due-diligence`, `fundability`, `refusals`, `exit-nav`, `proof-of-reserves`, …
- **53 academy pages** (`landing/src/pages/academy/**`): `/academy` learning journey (`01…20` + capstone + certificate + playground) **and** `/academy/onboarding/**` (the invite-gated real-money onboarding app + `[module]` dynamic route + certificate + `privacy`).
- **Design system:** canonical `SiteHeader`/`SiteFooter` in `Layout.astro`; Console tokens (`docs/SITE_DESIGN_SYSTEM.md`).
- **Live data path:** `DashboardLive.jsx` island polls `api.earn-defi.com` (~15s). Build-time static fallback = `landing/src/data/track_snapshot.json` (regenerated by the daily cycle).

## Backend / API map — `spa_core/api/`

- `server.py` — FastAPI app (port 8765). Mounts **17 routers** (`spa_core/api/routers/`) + the **Academy sub-app at `/academy`** (fail-safe try/except mount).
- Public endpoints incl. `/api/v1/golive`, `/api/live/*`, `/api/rates-desk/*`, `/api/refusal`, `/api/tier1/packages`, `/api/tournament`, `/academy/*`.
- **`spa_core/` is the entire product/runtime brain** (adapters, strategies, tournament, risk, paper_trading, strategy_lab, monitoring, governance, dr, academy). **stdlib-only runtime** by design (FastAPI/argon2/etc. are the documented exceptions for API/academy/cabinet).

## Duplication / fragmentation observations (document only — do NOT refactor)

1. **Academy tests appear in BOTH `spa_core/tests/` and (historically) other locations** — the CI-path confusion (`tests/` vs `spa_core/tests/`) suggests possible duplicate test trees. **Verify in PHASE 12.**
2. **Many `scripts/*deploy*`/`*push*` helpers overlap** (see AUDIT_03) — likely pre-API-push legacy alongside the canonical `push_to_github_batch.py`.
3. **`spa_core/` is large + intentionally so** — 100+ analyzer files are DISTINCT analyzers (a prior audit confirmed collapsing them would drop coverage). **Do NOT "consolidate" the analyzer family** — that is canonical breadth, not fragmentation.
4. **Two frontends (`landing/` + `cabinet/`)** are distinct products, not duplicates.
5. **Long files that are canonical (keep long):** `CLAUDE.md`, `spa_core/api/server.py`, `spa_core/risk/policy.py`, `spa_core/paper_trading/cycle_runner.py`. Do not split.

**Verdict:** the app is well-structured but sprawling; the real duplication risk is in **deploy scripts** and **possible duplicate test trees**, not the product code. No refactor recommended before PHASE 12 confirms references.
