## Что не так

- Токен `ghp_lXtPf…df2kN31r` лежит в plaintext в **77 файлах** на диске (`push_*.html`, `.bak*`) и в теле scheduled-task.
- Это PAT к репозиторию **money-moving** DeFi-системы. Он скомпрометирован и **не отозван 76 циклов**.
- Санкционированный заданием способ пуша (`push_v*.html → http://localhost:8765 → Chrome`) каждый раз создаёт **новую копию** утёкшего секрета. Автономно это делать нельзя.
- Канал `localhost:8765` из песочницы недоступен (`curl → 000`), так что пуш этим способом всё равно невозможен.

## Что делать (по порядку, ~10 минут)

1. **Отозвать токен.** GitHub → Settings → Developer settings → Personal access tokens → найти `ghp_lXtPf…` → **Revoke**. Делать это первым — пока токен жив, утечка активна.

2. **Выпустить новый PAT** с нужными scope (`repo`; для пуша workflow-файлов также `workflow`). **Не** вставлять его в task-файл, HTML или любой файл репозитория.

3. **Положить новый токен в Keychain** (вводится интерактивно, не попадает в историю shell):
   ```bash
   security add-generic-password -a "$USER" -s SPA_GITHUB_TOKEN -w
   ```
   *(или на время сессии: `export GITHUB_TOKEN=ghp_…` — только в env, не в файлы.)*

4. **Пушить через безопасный скрипт** (`secure_git_push.sh`, добавлен в этом цикле — токен читается из env/Keychain, в файлы не пишется, утёкший токен жёстко отвергается):
   ```bash
   cd ~/Documents/SPA_Claude
   ./secure_git_push.sh            # текущая ветка
   ./secure_git_push.sh main       # конкретная ветка
   ```

5. **Удалить файлы с утёкшим токеном** (после ротации, по вашему подтверждению — оркестратор не удаляет автономно):
   ```bash
   cd ~/Documents/SPA_Claude
   grep -rl "ghp_REVOKED_LEAKED_N31r" . | xargs rm -f   # проверьте список перед запуском
   ```

6. **Переписать scheduled-task** (`SKILL.md`): убрать PAT из тела; список спринтов SPA-V326…V332 устарел (всё реализовано); заменить правило «status pass запрещён» на «работать только при наличии реальной незаблокированной задачи»; заменить push-механизм на `./secure_git_push.sh`.
